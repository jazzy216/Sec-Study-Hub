Security controls, cryptography basics, authentication, PKI, and key concepts

Control Types (by Function)

  • Preventive - Stop incident before it occurs (firewall, lock)
  • Detective - Identify incident in progress (IDS, CCTV)
  • Corrective - Fix after incident (patch, backups)
  • Deterrent - Discourage attacks (warning signs, lighting)
  • Compensating - Alternative when primary control unavailable
  • Directive - Direct behavior (policies, training)

CIA Triad + Extensions

  • Confidentiality - Only authorized access (encryption, ACLs)
  • Integrity - Data not modified (hashing, digital sigs)
  • Availability - Systems accessible when needed (HA, backups)
  • Non-repudiation - Cannot deny action (digital signatures, logs)
  • Authentication - Verify identity
  • Authorization - What you can do after auth

Authentication Factors

  • Something you know - Password, PIN
  • Something you have - Smart card, token, phone
  • Something you are - Biometrics (fingerprint, iris)
  • Somewhere you are - Geolocation, IP range
  • Something you do - Behavioral (typing pattern)
MFA = 2+ different factor types. Same type twice = not MFA.

Cryptography Basics

  • Symmetric - Same key encrypt/decrypt (AES, 3DES). Fast. Key distribution problem.
  • Asymmetric - Public/private key pair (RSA, ECC). Slower. Solves key distribution.
  • Hashing - One-way, fixed output. Not encryption. (SHA-256, MD5 weak)
  • Hybrid - Asymmetric to exchange symmetric key. Used in TLS.

PKI Concepts

  • CA - Certificate Authority, issues/signs certs
  • Root CA - Top of trust chain, offline recommended
  • Intermediate CA - Issues end-entity certs, online
  • CRL - Certificate Revocation List (batch)
  • OCSP - Online Certificate Status Protocol (real-time)
  • CSR - Certificate Signing Request
  • Wildcard cert - *.domain.com (one domain, all subdomains)
  • SAN cert - Multiple different domains

Zero Trust Principles

  • Never trust, always verify
  • Assume breach — segment everything
  • Least privilege access at all times
  • Verify explicitly — use all data signals (ID, location, device health)
  • Micro-segmentation — limit blast radius
  • Continuous monitoring and validation

Key Terms

  • AAA - Authentication, Authorization, Accounting
  • SSO - Single Sign-On
  • SAML - XML-based SSO (enterprise)
  • OAuth 2.0 - Authorization delegation
  • OpenID Connect - Auth layer on OAuth
  • RADIUS - AAA for network access (UDP 1812/1813)
  • TACACS+ - Cisco AAA, TCP 49, full encryption
  • Kerberos - Ticket-based auth (port 88)

Domain 2: Threats, Vulnerabilities & Mitigations 22%

Attack types, malware, social engineering, vulnerability management

Social Engineering Attacks

  • Phishing - Mass email luring victims
  • Spear phishing - Targeted phishing (specific person)
  • Whaling - Phishing targeting executives (C-suite)
  • Vishing - Voice phishing (phone calls)
  • Smishing - SMS phishing
  • Pretexting - Fabricated scenario to extract info
  • Baiting - Leaving infected USB drives
  • Quid pro quo - Something for something (fake IT help)
  • Tailgating/Piggybacking - Following into restricted area
  • Business Email Compromise (BEC) - Impersonate exec, wire fraud

Malware Types

  • Virus - Attaches to file, needs user to run
  • Worm - Self-replicating, spreads without user (network)
  • Trojan - Disguised as legit software
  • Ransomware - Encrypts data, demands ransom
  • Spyware - Collects info covertly
  • Adware - Unwanted ads
  • Rootkit - Hides malware, root/kernel level
  • Keylogger - Records keystrokes
  • Backdoor/RAT - Persistent remote access
  • Fileless malware - Lives in memory/registry, no file on disk
  • Botnet - Network of infected machines (zombies)
  • Logic bomb - Triggers on condition (date, event)

Network Attacks

  • DoS - Overwhelm with traffic from one source
  • DDoS - Distributed, multiple sources (botnet)
  • Smurf - Amplified ICMP broadcast
  • SYN flood - Exploit TCP 3-way handshake
  • Ping of Death - Oversized ICMP packet
  • MITM - Intercept communications
  • ARP poisoning - Fake ARP, redirect traffic (Layer 2 MITM)
  • DNS poisoning - Corrupt DNS cache
  • SSL stripping - Downgrade HTTPS to HTTP
  • Replay attack - Reuse captured valid data
  • On-path attack - New name for MITM

Application Attacks

  • SQLi - Inject SQL into input fields. Mitigation: parameterized queries
  • XSS - Inject scripts into web page. Types: stored, reflected, DOM
  • CSRF - Trick user browser to send unauthorized request
  • SSRF - Force server to make internal requests
  • Directory traversal - ../ to access unauthorized files
  • Buffer overflow - Write beyond allocated memory
  • IDOR - Insecure Direct Object Reference
  • Race condition - TOCTOU (Time-of-Check/Time-of-Use)
  • Injection - SQL, LDAP, command, XML injection

Cryptographic Attacks

  • Birthday attack - Exploit hash collisions
  • Rainbow table - Precomputed hash lookups. Mitigate: salting
  • Brute force - Try all combinations
  • Dictionary attack - Try common words/passwords
  • Pass-the-hash - Reuse captured hash without cracking
  • Downgrade attack - Force weaker cipher/protocol
  • Side-channel - Exploit physical info (timing, power)
  • Padding oracle - CBC mode vulnerability

Vulnerability Concepts

  • Zero-day - No patch available yet
  • CVE - Common Vulnerabilities & Exposures (ID system)
  • CVSS - Common Vulnerability Scoring System (0-10)
  • NVD - National Vulnerability Database
  • Vulnerability scan - Non-intrusive, find known vulns
  • Penetration test - Active exploitation attempt
  • Bug bounty - Reward for responsible disclosure
  • Responsible disclosure - Notify vendor before public
  • Full disclosure - Release publicly without notifying vendor

Threat Intelligence

  • OSINT - Open-source intelligence (public data)
  • IoC - Indicator of Compromise (IP, hash, domain)
  • IoA - Indicator of Attack (TTPs, behavior)
  • STIX - Structured Threat Info eXpression (format)
  • TAXII - Transport protocol for STIX data
  • MITRE ATT&CK - Framework of adversary TTPs
  • TTP - Tactics, Techniques, Procedures
  • Threat hunting - Proactive search for hidden threats

Attack Frameworks & Kill Chain

Cyber Kill Chain (Lockheed Martin): Recon → Weaponize → Deliver → Exploit → Install → C2 → Actions on Objectives

MITRE ATT&CK Tactics (in order): Recon, Resource Dev, Initial Access, Execution, Persistence, Privilege Escalation, Defense Evasion, Credential Access, Discovery, Lateral Movement, Collection, C2, Exfiltration, Impact

Kill Chain mnemonic: Really Wicked Demons Eat Innocent Cats Alive = Recon, Weaponize, Deliver, Exploit, Install, C2, Actions

Domain 3: Security Architecture 18%

Network security, cloud, virtualization, secure infrastructure design

Network Segmentation

  • DMZ - Demilitarized zone; public-facing servers between two firewalls
  • VLAN - Virtual LAN; logical segmentation (Layer 2)
  • Micro-segmentation - Fine-grained, per-workload; used in SDN/cloud
  • Air gap - Physical isolation; no network connection
  • Extranet - Private network extended to partners
  • Intranet - Internal-only network
  • East-West traffic - Lateral (server to server)
  • North-South traffic - In/out of data center

Firewall Types

  • Packet filtering - Layer 3/4; checks IP/port; stateless
  • Stateful - Tracks connection state; most common
  • Application/Proxy - Layer 7; deep inspection; understands protocols
  • NGFW - Next-gen; combines stateful + IPS + app awareness + user ID
  • WAF - Web Application Firewall; HTTP/S traffic; stops SQLi, XSS
  • Host-based FW - Software on the endpoint
  • UTM - Unified Threat Mgmt; all-in-one (SMB)

IDS vs IPS

FeatureIDSIPS
ActionAlert onlyBlock + alert
PlacementOut-of-band (SPAN port)Inline
RiskFalse negative = missFalse positive = block legit
  • Signature-based - Match known patterns; low FP, misses new attacks
  • Anomaly/heuristic - Baseline deviation; catches new, higher FP
  • Behavior-based - Watch actions over time

VPN Types

  • Remote access VPN - Client to site (IPSec or SSL/TLS)
  • Site-to-site VPN - Gateway to gateway
  • Split tunnel - Only some traffic goes through VPN
  • Full tunnel - All traffic through VPN
  • SSL VPN - Uses TLS; browser or client; port 443
  • IPSec - Tunnel or transport mode; AH or ESP
  • Always-on VPN - Connects automatically (modern Zero Trust)

Cloud Deployment Models

  • Public - Multi-tenant, internet (AWS, Azure, GCP)
  • Private - Single org, on-prem or hosted
  • Hybrid - Mix of public + private
  • Community - Shared by orgs with common interests
  • Multi-cloud - Using 2+ cloud providers

Virtualization & Containers

  • Hypervisor Type 1 - Bare metal (VMware ESXi, Hyper-V)
  • Hypervisor Type 2 - Runs on OS (VirtualBox, VMware Workstation)
  • VM escape - Attack breaks out of VM to host
  • Container - Lightweight, shares OS kernel (Docker)
  • Container escape - Break out of container
  • Serverless - No server management; event-driven (Lambda)
  • Snapshot - Point-in-time copy of VM state

Load Balancing & HA

  • Active-Active - Both nodes handle traffic simultaneously
  • Active-Passive - One active, one standby; failover only
  • Round-robin - Rotate requests evenly
  • Least connection - Send to server with fewest connections
  • RTO - Recovery Time Objective (how long to recover)
  • RPO - Recovery Point Objective (how much data loss tolerable)
  • MTTR - Mean Time To Repair
  • MTBF - Mean Time Between Failures

Domain 4: Security Operations 28%

Incident response, digital forensics, monitoring, IAM, endpoint security

Incident Response Process

  • 1. Preparation - Policies, tools, training, IR plan
  • 2. Identification - Detect and confirm incident
  • 3. Containment - Limit spread (short-term + long-term)
  • 4. Eradication - Remove malware, close attack vector
  • 5. Recovery - Restore systems, verify normal operations
  • 6. Lessons Learned - Post-incident review, improve defenses
PICERL: Preparation, Identification, Containment, Eradication, Recovery, Lessons

Digital Forensics Principles

  • Order of volatility - Collect most volatile first: CPU/RAM → Swap → Disk → Logs → Archived media
  • Chain of custody - Document who handled evidence and when
  • Legal hold - Preserve data for litigation
  • Write blocker - Prevent modifying evidence disk
  • Forensic copy - Bit-for-bit image (dd, FTK Imager)
  • Hash verification - MD5/SHA to prove integrity of copy
  • Locard's principle - Every contact leaves a trace

Log & Monitoring Tools

  • SIEM - Security Info & Event Mgmt; aggregate, correlate logs (Splunk, QRadar)
  • SOAR - Security Orchestration, Automation & Response; automate playbooks
  • Syslog - Standard log forwarding (UDP 514)
  • NetFlow - Network traffic metadata (no payload)
  • PCAP - Packet capture (full content); Wireshark
  • NXlog / Beats - Log shippers
  • EDR - Endpoint Detection & Response; behavioral monitoring on endpoint
  • XDR - Extended DR; correlates across endpoint, network, cloud

Identity & Access Management

  • DAC - Discretionary: owner sets permissions (NTFS)
  • MAC - Mandatory: system enforces labels (Top Secret, etc.)
  • RBAC - Role-based: permissions by job role
  • ABAC - Attribute-based: dynamic rules on attributes
  • PBAC - Policy-based: similar to ABAC
  • Rule-based AC - Static rules (firewall ACLs)
  • Least privilege - Minimum needed to do job
  • Separation of duties - Split critical tasks among multiple people
  • Dual control - Two people required simultaneously
  • Job rotation - Rotate roles to detect fraud

Account Management

  • Provisioning - Create/grant accounts
  • Deprovisioning - Remove when employee leaves (critical!)
  • Recertification - Periodic review of access rights
  • Privileged accounts - Admin/root; monitor closely
  • PAM - Privileged Access Management; vault for admin creds
  • Service accounts - Non-human; applications
  • Shared accounts - Avoid; no individual accountability
  • Time-of-day restrictions - Limit when accounts can log in

Endpoint Security

  • AV/Anti-malware - Signature + heuristic detection
  • EDR - Continuous monitoring, behavioral analysis
  • DLP - Data Loss Prevention; stop data exfiltration
  • HIDS - Host-based IDS
  • FIM - File Integrity Monitoring (tripwire)
  • Application whitelisting - Only approved apps can run
  • Patch management - Keep systems updated
  • Mobile MDM/MAM - Mobile Device/App Management
  • Sandboxing - Isolate unknown code

Secure Data Destruction

  • Degaussing - Strong magnetic field; destroys magnetic media
  • Shredding/Pulverizing - Physical destruction
  • Overwriting - Write patterns over data (DoD 5220.22-M)
  • Crypto-shredding - Destroy encryption key (cloud/SSD best practice)
  • Incineration - Burn it
  • Sanitization vs Destruction - Sanitize = reuse possible; Destroy = not reusable

Backup & Recovery

  • Full backup - Everything; slowest; fastest restore
  • Incremental - Changes since last backup (any type); slowest restore
  • Differential - Changes since last FULL; faster restore than incremental
  • 3-2-1 Rule - 3 copies, 2 different media types, 1 offsite
  • Snapshot - Point-in-time (VM/storage)
  • Cold site - Empty facility; slow to activate
  • Warm site - Some equipment pre-installed
  • Hot site - Fully operational duplicate; immediate failover

Common Vulnerability Scanning

  • Credentialed scan - Uses login; sees inside OS; more thorough
  • Non-credentialed - External view; like attacker
  • Active scan - Sends traffic to target
  • Passive scan - Listen only; no traffic sent
  • Agent-based - Software on endpoint, real-time
  • False positive - Reports vuln that doesn't exist
  • False negative - Misses real vuln (worse!)
  • Nessus, Qualys, OpenVAS - Common scanners

Pen Testing Phases

  • 1. Planning - Scope, rules of engagement, legal authorization
  • 2. Recon - Passive (OSINT) and active
  • 3. Scanning - Enumerate targets, find vulns
  • 4. Exploitation - Attack found vulnerabilities
  • 5. Post-exploitation - Pivot, escalate, persist
  • 6. Reporting - Document findings, recommendations
  • Black box - No prior knowledge
  • Gray box - Some knowledge (user-level)
  • White box - Full knowledge (source code, network maps)

Domain 5: Security Program Management & Oversight 20%

Risk management, compliance, policies, privacy, and frameworks

Risk Management

  • Risk = Threat × Vulnerability × Impact
  • Asset value (AV) - What the asset is worth
  • Exposure factor (EF) - % of asset lost if risk occurs
  • Single Loss Expectancy (SLE) = AV × EF
  • Annual Rate of Occurrence (ARO) - How often per year
  • Annual Loss Expectancy (ALE) = SLE × ARO
  • Residual risk - Risk remaining after controls applied
  • Inherent risk - Risk before any controls
ALE = SLE × ARO — memorize this formula cold!

Risk Response Strategies

  • Avoid - Stop the risky activity entirely
  • Transfer - Shift to third party (insurance, outsource)
  • Mitigate/Reduce - Apply controls to reduce likelihood or impact
  • Accept - Acknowledge risk, no action (residual risk)
  • Reject/Ignore - Not a valid strategy! (wrong answer)

Security Frameworks

  • NIST CSF - Identify, Protect, Detect, Respond, Recover
  • NIST SP 800-53 - Federal security controls catalog
  • ISO 27001 - International ISMS standard
  • CIS Controls - 18 prioritized security controls
  • COBIT - IT governance framework
  • ITIL - IT service management
  • SOC 2 - Service org controls (Trust Services)
  • CSA CCM - Cloud Security Alliance controls

Data Classification

  • Government: Top Secret > Secret > Confidential > Unclassified
  • Commercial: Restricted > Confidential > Internal > Public
  • Data owner - Determines classification and access
  • Data custodian - Manages and protects data (IT)
  • Data steward - Ensures quality and proper use
  • Data processor - Third party processing data (GDPR term)
  • Data controller - Determines purpose of processing (GDPR term)

Privacy Concepts

  • PII - Personally Identifiable Information
  • PHI - Protected Health Information
  • Data minimization - Collect only what's needed
  • Right to be forgotten - GDPR erasure right
  • Tokenization - Replace sensitive data with token (PCI)
  • Anonymization - Remove all identifying info (irreversible)
  • Pseudonymization - Replace ID with alias (reversible)
  • Privacy by design - Build privacy in from start

Third-Party Risk

  • Vendor assessment - Evaluate before onboarding
  • SLA - Service Level Agreement; uptime, response guarantees
  • MOU - Memorandum of Understanding; intent, non-binding
  • MOA - Memorandum of Agreement; binding
  • MSA - Master Service Agreement; framework contract
  • NDA - Non-Disclosure Agreement
  • ISA - Interconnection Security Agreement; between orgs sharing network
  • BPA - Business Partnership Agreement
  • Supply chain attack - Compromise via trusted vendor (SolarWinds)
